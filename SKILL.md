| 命令 | 对应三省 | 技能名称 | 作用 |
|------|---------|---------|------|
| **/spec** | 中书省 | spec-driven-development | 需求定义、规格说明 |
| **/plan** | 门下省 | planning-and-task-breakdown | 规划分解、任务拆解 |


## 标准执行流程 v2.0

```
  ┌─────────┐
  │  中书省  │ → 宰相下达指令，明确需求规格  →  /spec
  └────┬────┘
       │
  ┌────▼────┐
  │  门下省  │ → 审核任务规范，分解执行计划  →  /plan
  └────┬────┘
       │
  ┌────▼────┐
  │  尚书省  │ → 六部按分工执行，增量构建  →  /build
  └────┬────┘
       │
  ┌────▼────┐
  │  门下省  │ → 测试验证成果，复核质量     →  /test + /review
  └────┬────┘
       │
  ┌────▼────┐
  │  中书省  │ → 宰相验收，发布交付        →  /ship
  └─────────┘
```

### 详细步骤

#### Step 1: 中书省 - 需求定义 (/spec)
- 宰相明确任务目标和输出标准
- 使用 `/spec` 命令制定完整需求规格
- 确定参与龙虾、时间节点、验收标准

#### Step 2: 门下省 - 规划审核 (/plan)
- 工作流优化师审核任务规范
- 使用 `/plan` 分解为可执行子任务
- 确定依赖关系和资源配置

#### Step 3: 尚书省 - 执行构建 (/build)
- 各专业龙虾按分工执行
- 使用 `/build` 增量实现，小步快跑
- 每完成一个子任务及时汇报进度

#### Step 4: 门下省 - 测试复核 (/test + /review)
- 使用 `/test` 验证成果质量
- 使用 `/review` 进行专业审核
- 发现问题立即退回修正

#### Step 5: 中书省 - 发布交付 (/ship)
- 宰相验收最终成果
- 使用 `/ship` 发布交付
- 归档知识库，更新多维表格

## 异常处理机制

### 1. 龙虾无响应
- 5分钟未回复 → 门下省再次@提醒
- 10分钟未响应 → 上报宰相协调
- 15分钟未响应 → 宰相重新分配任务

### 2. 产出质量不达标
- 门下省审核不通过 → 标注问题点退回重作
- 重作不超过2次 → 仍不合格则宰相介入

### 3. 任务超时
- 预计超时提前5分钟预警
- 超时未完成 → 宰相协调追加资源
