## 技能描述

本技能让新龙虾能够：
1. ✅ 接收主龙虾分配的任务
2. ✅ 通过 sessions_spawn 创建子代理会话执行任务
3. ✅ 通过 openid 直接@每个龙虾发送任务通知
4. ✅ 创建飞书文档并记录链接
5. ✅ 更新飞书多维表格
6. ✅ 汇总所有成果交付

---

## 核心能力

### 1. 任务接收与理解
- 理解主龙虾的任务分配指令
- 识别任务编号、任务类型、输出要求
- 识别截止时间和依赖关系

### 2. 任务执行（sessions_spawn）
```python
sessions_spawn(
    agentId="main",
    cleanup="keep",
    cwd="/Users/laihehuo/.openclaw/workspace-main/agents/main",
    label="龙虾 X 号 - 任务类型",
    mode="run",
    runtime="subagent",
    task="任务描述..."
)
```

### 3. 飞书通知（openid 私聊@）

**方式 1：通过 openid 私聊发送（推荐）**

```python
feishu_im_user_message(
    action="send",
    msg_type="text",
    receive_id="ou_xxx",  # 龙虾的 openid
    receive_id_type="open_id",
    content='{"text":"🦞 龙虾 X 号，收到请执行！\\n\\n【任务】..."}'
)
```

**方式 2：通过飞书群@发送（跨应用备用）**

```python
# 如果 openid 私聊失败（跨应用限制），改用飞书群@
feishu_im_user_message(
    action="send",
    msg_type="text",
    receive_id="oc_xxx",  # 群聊 ID
    receive_id_type="chat_id",
    content='{"text":"🦞 <at id=\\"ou_xxx\\"></at> 龙虾 X 号，收到请执行！..."}'
)
```

**注意**：
- 优先使用 openid 私聊，确保每只龙虾都能收到单独指令
- 如遇跨应用限制（open_id cross app 错误），改用飞书群@方式
- 龙虾 7 号（云端龙虾）通常需要使用飞书群@方式

### 4. 飞书文档创建
```python
feishu_create_doc(
    title="🦞 任务名称 - 日期",
    markdown="[Markdown 内容]",
    folder_token="[文件夹 token]"
)
```

## 执行流程

```
1. 接收用户指令
    ↓
2. 拆解为 6-7 个子任务
    ↓
3. sessions_spawn 创建子代理会话（执行）
    ↓
4. feishu_im_user_message 通过 openid 私聊@每个龙虾（通知）
   - 优先使用 openid 私聊（receive_id_type="open_id"）
   - 如遇跨应用限制，改用飞书群@（receive_id_type="chat_id"）
   - 龙虾 7 号（云端龙虾）通常使用飞书群@方式
    ↓
5. 各子代理并行执行任务
    ↓
6. 子代理完成后：
   - 自动推送成果到指挥中心群
   - 创建飞书文档
   - 按照指定格式回复：
     ✅ 龙虾 X 号完成
     📄 本地文件：[文件路径]
     🔗 飞书文档：[飞书文档链接]
    ↓
7. **依赖自动通知**：当前龙虾完成后，必须以飞书用户身份通知下一步依赖的龙虾
   - 格式：`🦞 @ou_xxx 依赖任务已完成，请开始执行你的任务`
   - 确保流水式任务能够自动流转，不需要主龙虾介入
    ↓
8. 主助手收集所有成果
    ↓
9. 龙虾 7 号创建汇总飞书文档
    ↓
10. 更新飞书多维表格（包含所有产出链接）
    ↓
11. 交付完整成果给用户
```

## 依赖通知规则

### 规则说明
当任务存在依赖关系（A 完成 → B 才能开始），**A 完成后必须主动通知 B**：

```python
# A 完成后，A 需要以飞书用户身份通知 B
feishu_im_user_message(
    action="send",
    msg_type="text",
    receive_id="B's-openid",
    receive_id_type="open_id",
    content='{"text":"🦞 @ou_xxx 【TASK-XXXXXX】依赖任务已完成，请开始执行你的任务\n\n任务链接：[飞书文档链接]"}'
)
```

### 标准依赖通知格式
```
🦞 @ou_xxx 

【TASK-XXXXXX】依赖任务已完成

已完成：[任务名称]，包含具体产出内容
飞书文档：[**必须填入实际飞书文档链接**]

现在你可以开始执行任务，请继续。

按照规则：
1. 创建本地任务编排文件
2. 配置心跳任务监控
3. 完成后创建飞书文档 → 通知下一步依赖龙虾

期待你的交付！
```

### 强制要求
- ✅ **通知中必须包含已完成任务的飞书文档链接**，不能省略
- ✅ 链接必须是可点击的飞书分享链接
- ✅ 接收方才能直接点击查看内容
- ✅ 如果省略链接，接收方无法开始工作

## 常见问题

### Q1：新龙虾如何学会这个技能？

### Q3：飞书文档创建失败怎么办？

A：检查飞书授权是否完成，Markdown 内容格式是否正确。

### Q4：多维表格记录失败怎么办？

A：检查表格字段是否匹配，日期格式是否正确（yyyy/MM/dd）。

### Q5：如何添加新龙虾？

A：在 `config/lobster-team.md` 中添加新龙虾信息，获取 OpenID，在飞书群中测试。

### Q6：openid 私聊失败怎么办？

A：如遇 `open_id cross app` 错误（跨应用限制），改用飞书群@方式：
```python
feishu_im_user_message(
    action="send",
    msg_type="text",
    receive_id="oc_xxx",  # 群聊 ID
    receive_id_type="chat_id",
    content='{"text":"🦞 <at id=\\"ou_xxx\\"></at> 龙虾 X 号，收到请执行！..."}'
)
```

### Q7：飞书@语法是什么？

A：正确的飞书@语法：
- **格式**：`<at id="ou_xxx"></at>`
- **示例**：`🦞 <at id="ou_f243f998e0eaf197cb6eebb82a324ece"></at> 龙虾 1 号，收到请执行！`
- **注意**：需要使用飞书群聊（chat_id），不能用于私聊
